# Becoming AI Native

An open, evolving playbook for agencies going AI-native. Tracked problems, documented fixes, compounding knowledge. Built and maintained by [reverseBits](https://reversebits.tech). Contributions welcome from any agency, studio, or indie operator.

---

## Quickstart (60 seconds)

> **If you only read one section, read this one.**

Hit a problem?
→ [Open an issue](../../issues/new/choose). Pick the "Problem report" template. Fill 3 fields. Done.

Want to share a tool, workflow, or learning?
→ Use one of the other templates. Same page.

Looking for a fix someone already figured out?
→ Browse [`/docs`](./docs) — every solved problem has a clean, linkable doc there.

Just want to chat or ask a question?
→ [Discussions](../../discussions).

That's the whole contributor loop. Stop here if you're filing or reading. Everything below is for maintainers.

---

## What this repo is

Not cheatsheets. Not hype threads. A tracked, searchable, versioned knowledge base of the real problems agencies hit adopting Claude, MCPs, skills, plugins, and agentic workflows — and the fixes, tools, and workflows that actually worked.

Two layers:

- **Issues** — raw problems, evidence trails, in-flight investigations.
- **`/docs`** — promoted, curated fixes. Stable URLs. Shareable. This is the compounding asset.

A problem starts as an issue. When it's solved, the fix lands as a doc in `/docs` via PR, and the issue closes with a link to the doc.
